Flexbox vs Grid

Two approaches to placing 5 equal-width cards inside a 12-column Bootstrap layout — with different content heights in each card.

Method 1

Flexbox approach

Use display: flex with flex-wrap: wrap on the row container. Each card gets flex: 1 1 calc(20% - 13px) so they share space equally. Cards in the same row will auto-stretch to match the tallest sibling — but only within that row.

Method 2

CSS Grid — fixed 5 cols (needs media queries)

Uses repeat(5, 1fr) — always exactly 5 columns regardless of screen size. Needs @media queries to collapse on mobile.

Method 3 ✦ Best — Zero @media

auto-fit + minmax — fully responsive, one line

Replace repeat(5, 1fr) with repeat(auto-fit, minmax(180px, 1fr)). The browser itself decides how many columns fit. When space is tight, cards wrap to the next row and the remaining cards stretch to fill the full width. Zero media queries needed.

/* ── auto-fit: ZERO media queries needed ── */

.grid-auto-fit {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(180px, 1fr));
  gap: 16px;
}

/*  auto-fit   → collapses empty tracks, so remaining
               cards GROW and fill the full row width.
    minmax()   → card is at least 180px wide.
               once there's no room for another 180px
               column, that card wraps to a new row.
    1fr        → all cards share leftover space equally. */

/* That's it — no @media at all! Resize the window ↔ */
Method 4

auto-fill — same but cards don't stretch

auto-fill works identically to auto-fit when all tracks are occupied. The difference appears when cards wrap: auto-fill keeps invisible ghost columns, so the last row cards keep their natural minmax width instead of stretching. Use this when you don't want the last row to look different.

Side-by-side comparison

Criterion Flexbox CSS Grid
Dimension 1D — controls one axis at a time (row OR column) 2D — controls rows AND columns simultaneously
Equal widths ✓ with flex: 1 1 calc(20% - gap) — a little math needed ✓ trivially with repeat(5, 1fr)
Equal heights per row ✓ auto-stretch in same row ✓ auto-stretch across all rows
Gap consistency ✓ gap works, but calc() needed for width ✓ gap handled entirely by grid — no math
Responsive control Relies on min-width + flex-wrap wrapping Change one property: grid-template-columns
Item reordering / spanning Hard — items flow linearly Easy — use grid-column: span 2 etc.
Browser support Excellent (IE11+) Excellent (all modern browsers; IE11 needs prefix)
Best for Navbars, toolbars, single-axis layouts, centering Card grids, dashboards, page sections, 2D layouts

🏆 Verdict — auto-fit + minmax is the ultimate one-liner

One CSS rule replaces all your media queries for card grids:

grid-template-columns: repeat(auto-fit, minmax(180px, 1fr));
The browser figures out column count automatically. Cards are at least 180px wide, share space equally via 1fr, and wrap naturally when the container gets narrow — no @media needed at all.

Use auto-fill instead if you want wrapped cards to stay at their natural minimum width rather than stretching to fill the last row.
